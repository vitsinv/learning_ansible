<h2 dir="auto"><a id="user-content-подготовка-к-выполнению" class="anchor" aria-hidden="true" href="#подготовка-к-выполнению"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Подготовка к выполнению</h2>
<ol dir="auto">
<li>Установите molecule: <code>pip3 install "molecule==3.4.0"</code></li>
<li>Соберите локальный образ на основе <a href="/netology-code/mnt-homeworks/blob/MNT-7/08-ansible-05-testing/Dockerfile">Dockerfile</a></li>
</ol>
<h2 dir="auto"><a id="user-content-основная-часть" class="anchor" aria-hidden="true" href="#основная-часть"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Основная часть</h2>
<p dir="auto">Наша основная цель - настроить тестирование наших ролей. Задача: сделать сценарии тестирования для kibana, filebeat. Ожидаемый результат: все сценарии успешно проходят тестирование ролей.</p>
<h3 dir="auto"><a id="user-content-molecule" class="anchor" aria-hidden="true" href="#molecule"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Molecule</h3>
<ol dir="auto">
<li>Запустите  <code>molecule test</code> внутри корневой директории elasticsearch-role, посмотрите на вывод команды.</li>
<li>Перейдите в каталог с ролью kibana-role и создайте сценарий тестирования по умолчаню при помощи <code>molecule init scenario --driver-name docker</code>.</li>
<li>Добавьте несколько разных дистрибутивов (centos:8, ubuntu:latest) для инстансов и протестируйте роль, исправьте найденные ошибки, если они есть.</li>
<li>Добавьте несколько assert'ов в verify.yml файл, для  проверки работоспособности kibana-role (проверка, что web отвечает, проверка логов, etc). Запустите тестирование роли повторно и проверьте, что оно прошло успешно.</li>
<li>Повторите шаги 2-4 для filebeat-role.</li>
<li>Добавьте новый тег на коммит с рабочим сценарием в соответствии с семантическим версионированием.</li>
</ol>
<h3 dir="auto"><a id="user-content-tox" class="anchor" aria-hidden="true" href="#tox"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Tox</h3>
<ol dir="auto">
<li>Запустите <code>docker run --privileged=True -v &lt;path_to_repo&gt;:/opt/elasticsearch-role -w /opt/elasticsearch-role -it &lt;image_name&gt; /bin/bash</code>, где path_to_repo - путь до корня репозитория с elasticsearch-role на вашей файловой системе.</li>
<li>Внутри контейнера выполните команду <code>tox</code>, посмотрите на вывод.</li>
<li>Добавьте файл <code>tox.ini</code> в корень репозитория каждой своей роли.</li>
<li>Создайте облегчённый сценарий для <code>molecule</code>. Проверьте его на исполнимость.</li>
<li>Пропишите правильную команду в <code>tox.ini</code> для того чтобы запускался облегчённый сценарий.</li>
<li>Запустите <code>docker</code> контейнер так, чтобы внутри оказались обе ваши роли.</li>
<li>Зайдти поочерёдно в каждую из них и запустите команду <code>tox</code>. Убедитесь, что всё отработало успешно.</li>
<li>Добавьте новый тег на коммит с рабочим сценарием в соответствии с семантическим версионированием.</li>
</ol>
<p dir="auto">После выполнения у вас должно получится два сценария molecule и один tox.ini файл в каждом репозитории. Ссылки на репозитории являются ответами на домашнее задание. Не забудьте указать в ответе теги решений Tox и Molecule заданий.</p>
<h2 dir="auto"><a id="user-content-необязательная-часть" class="anchor" aria-hidden="true" href="#необязательная-часть"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Необязательная часть</h2>
<ol dir="auto">
<li>Проделайте схожие манипуляции для создания роли logstash.</li>
<li>Создайте дополнительный набор tasks, который позволяет обновлять стек ELK.</li>
<li>В ролях добавьте тестирование в раздел <code>verify.yml</code>. Данный раздел должен проверять, что logstash через команду <code>logstash -e 'input { stdin { } } output { stdout {} }'</code>  отвечате адекватно.</li>
<li>Создайте сценарий внутри любой из своих ролей, который умеет поднимать весь стек при помощи всех ролей.</li>
<li>Убедитесь в работоспособности своего стека. Создайте отдельный verify.yml, который будет проверять работоспособность интеграции всех инструментов между ними.</li>
<li>Выложите свои roles в репозитории. В ответ приведите ссылки.</li>
</ol>
